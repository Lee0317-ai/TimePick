## 1. 功能改造清单

### 1.1 任务状态流转完善

**问题**: 点击"开始任务"后状态变为"尝试中"，但无法继续操作（没有"完成"按钮）

**改造内容**:
- 新增 `CompleteTodoDialog` 弹窗，支持完成任务时：
  - 选择完成状态（已完成/暂不尝试/已放弃）
  - 星级评分（1-5星）
  - 添加备注
  - 转换为资源库条目（可选）
- `LinkCard` 和 `LinkDetailView` 为 `trying` 状态添加"完成"按钮

**相关文件**:
- `src/components/TodoPage.tsx`
- `src/components/CompleteTodoDialog.tsx`

---

### 1.2 详情页修复

#### 1.2.1 按钮无效问题
**问题**: 详情页的"开始任务"按钮点击无响应

**原因**: `LinkDetailView` 是独立函数组件，无法访问父组件的 `handleStartTry` 等处理函数

**修复**: 将所需处理函数作为 props 传递给 `LinkDetailView`：
- `onStartTry`: 开始任务
- `onCompleteTry`: 完成任务（打开完成弹窗）
- `onOpenLink`: 打开链接
- `onResumeTry`: 继续尝试

#### 1.2.2 返回按钮问题
**问题**: 点击返回后回到主页而非列表

**原因**:
1. 返回按钮图标是 `ArrowRightCircle`（箭头朝右），与顶部"回主页"按钮相同
2. 原图标没有 `rotate-180`，用户误点顶部按钮

**修复**:
- 改用 `ArrowLeft` 图标
- 添加文字"返回列表"明确区分

**相关文件**:
- `src/components/TodoPage.tsx`

---

### 1.3 文件夹下拉选择修复

**问题**: 完成任务时选择文件夹下拉无响应/白屏

**原因**:
1. `DropdownMenu` 在 `Dialog` 内部有焦点捕获冲突
2. `folders` 表查询了不存在的 `icon` 字段导致报错
3. `SelectContent` 缺少必要的定位属性

**修复**:
1. 用 `Select` 组件替换 `DropdownMenu`
2. 修正查询字段：移除 `icon`，只查询 `id, name`
3. 添加 `position="popper"` 和 `modal={false}` 属性
4. 修正 `SelectItem` 空值问题（空字符串作为 value 会导致问题）

**相关文件**:
- `src/components/CompleteTodoDialog.tsx`

---

### 1.4 Toast 语法错误修复

**问题**: 多处操作后出现白屏

**原因**: 项目使用 `sonner` 库，但多处代码用了 shadcn/ui 的 `useToast` 对象语法

**受影响的处理函数**:
- `handleStartTry` - 开始任务
- `handleDelete` - 删除任务
- `CompleteTodoDialog.handleSave` - 完成任务

**修复**: 统一改为 sonner 语法：
```typescript
// 错误 ❌
toast({ title: "消息", description: "详情", variant: "destructive" })

// 正确 ✅
toast.success("消息", { description: "详情" })
toast.error("错误消息")
```

**相关文件**:
- `src/components/TodoPage.tsx`
- `src/components/CompleteTodoDialog.tsx`

---

### 1.5 "继续尝试"功能

**问题**: 标记为"暂不尝试"的任务无法恢复

**改造内容**:
- 新增 `handleResumeTry` 处理函数
- 将 `deferred` 状态改回 `trying`，重置开始时间
- 自动打开链接
- `LinkCard` 和 `LinkDetailView` 为 `deferred` 状态添加"▶️ 继续尝试"按钮
- "打开链接"按钮现在也支持 `deferred` 状态

**相关文件**:
- `src/components/TodoPage.tsx`
