## 🎯 阶段 2：后端服务（Supabase Edge Functions）

### T-201: 实现网页内容抓取服务
- **类型**: 后端
- **优先级**: P0
- **预估**: 4 小时
- **前置依赖**: T-101, T-102
- **执行**:
  - 创建 Edge Function: `fetch-webpage-metadata`
  - 功能：
    - 接收 URL 参数
    - 使用 `fetch` 获取网页内容
    - 解析 HTML，提取 `title`、`meta description`、`meta keywords`
    - 返回 JSON 结果
  - 错误处理：
    - 超时（10 秒）
    - 网络错误
    - 不支持的 Content-Type
- **验收**:
  - Edge Function 部署成功
  - 可以成功抓取常见网页（GitHub、博客、文档）
  - 错误情况下返回友好的错误信息

### T-202: 实现优先级计算服务
- **类型**: 后端
- **优先级**: P0
- **预估**: 6 小时
- **前置依赖**: T-201
- **执行**:
  - 创建 Edge Function: `calculate-priority`
  - 功能：
    - 接收参数：`url`、`title`、`description`、`user_id`、`tags`
    - **关键词分析**：
      - 定义高/中/低优先级关键词列表
      - 匹配标题和描述，计算基础分数
    - **学习相关性分析**：
      - 查询用户的 `learning_focus` 表
      - 匹配标题（完全匹配/部分匹配）
      - 应用权重计算
    - **标签调整**：
      - 检查特殊标签（🔥紧急、⭐必看等）
      - 应用标签加分/减分
    - 返回最终分数和优先级等级（high/medium/low）
  - 缓存优化：
    - 对相同 URL 的计算结果缓存 1 小时
- **验收**:
  - 优先级计算符合 PRD-002 规则
  - 单次计算耗时 < 500ms
  - 缓存命中时 < 50ms

### T-203: 实现批量优先级计算任务
- **类型**: 后端
- **优先级**: P0
- **预估**: 3 小时
- **前置依赖**: T-202
- **执行**:
  - 创建 Edge Function: `batch-calculate-priority`
  - 功能：
    - 接收 `link_ids` 数组
    - 遍历调用 `calculate-priority`
    - 并发处理（最多 5 个并发）
    - 批量更新数据库
  - 异步任务调度：
    - 使用 Supabase 的 `pg_cron` 或定时触发
    - 每日自动重新计算所有"未开始"的链接
- **验收**:
  - 批量处理 50 条链接耗时 < 30 秒
  - 异步任务正常执行

## ✅ 验收标准（整体）

所有任务完成后，以下功能应正常工作：

1. ✅ 用户可以快速添加单条待尝试链接
2. ✅ 用户可以从微信批量粘贴多条链接并导入
3. ✅ 系统自动计算优先级（基于关键词、学习相关性、标签）

7. ✅ 用户可以"开始尝试"并跟踪进度
8. ✅ 用户可以完成、评分、归档链接
9. ✅ 已完成的链接可以转化为正式资源
